## Layer Architecture

### 1. Monorepo (Root)

```
studymate-ai/
в”ңв”Җв”Җ apps/
в”Ӯ   в”ңв”Җв”Җ backend/          # NestJS API server
в”Ӯ   в””в”Җв”Җ frontend/         # Next.js 15 application
в”ңв”Җв”Җ packages/
в”Ӯ   в”ңв”Җв”Җ shared/           # Types, Zod schemas, constants
в”Ӯ   в”ңв”Җв”Җ db/               # Drizzle schema, migrations, seeds
в”Ӯ   в”ңв”Җв”Җ config-eslint/    # Shared ESLint configuration
в”Ӯ   в”ңв”Җв”Җ config-typescript/# Shared TypeScript configuration
в”Ӯ   в””в”Җв”Җ config-tailwind/  # Shared Tailwind CSS configuration
в”ңв”Җв”Җ docker/               # Docker Compose + Dockerfiles
в””в”Җв”Җ docs/                 # VitePress documentation site
```

**Orchestration:** Turborepo manages dependency graph, build caching, and parallel task execution across all packages.

```
pnpm dev    вҶ’ turbo runs dev in all apps/packages concurrently
pnpm build вҶ’ turbo builds in topological order (dependencies first)
pnpm test  вҶ’ turbo runs tests in parallel with dependency awareness
```

## Communication Patterns

### Synchronous REST
- Most CRUD operations (documents, conversations, quizzes, rooms)
- Request вҶ’ Response within 200-500ms
- Standard JSON payloads

### Server-Sent Events (SSE)
- Chat message endpoint (`POST /api/chat/message`)
- Single POST request returns a stream of events
- Events: `token` | `citation` | `done`
- Backend pushes tokens as Gemini generates them
- Frontend reads via `ReadableStream`

### WebSockets (Socket.IO)
- Study room real-time chat
- Room presence (who's online)
- Shared quiz sessions (future)
- Managed by NestJS `@nestjs/websockets` gateway

### Presigned URL Upload
- Frontend requests upload URL from backend
- Backend generates S3 presigned URL (valid for 5 min)
- Frontend uploads directly to S3
- Frontend notifies backend to process
- Avoids server-side file buffering

## Error Handling Strategy

```
All errors follow a consistent shape:
{
  "status": 400,
  "message": "Human-readable error description",
  "errors": [
    {
      "field": "title",
      "message": "Title is required"
    }
  ]
}

Error categories:
в”ңв”Җв”Җ 400 вҖ” Validation errors (Zod schema failures)
в”ңв”Җв”Җ 401 вҖ” Unauthorized (missing/invalid token)
в”ңв”Җв”Җ 403 вҖ” Forbidden (not owner of resource)
в”ңв”Җв”Җ 404 вҖ” Resource not found
в”ңв”Җв”Җ 409 вҖ” Conflict (duplicate, state mismatch)
в”ңв”Җв”Җ 413 вҖ” Payload too large (file exceeds limit)
в”ңв”Җв”Җ 429 вҖ” Rate limited
в””в”Җв”Җ 500 вҖ” Internal server error (with Sentry alert)
```
